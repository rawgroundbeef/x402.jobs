<objective>
Remediate HIGH-07 (missing recipient validation in `verifySolanaPayment`) and HIGH-10 (legacy `transfer` instructions accepted without mint check) in `routes/webhooks.ts`. These are interlocking fixes to the same function (`verifySolanaPayment`), so they ship together.

Purpose: Today the function only validates that *some* USDC transfer of the expected amount appears in the transaction. A compromised facilitator (or any party that can craft a parsed-tx structure) could reroute payment to a wallet they control. After this fix, the destination ATA must match the recipient's USDC ATA AND the mint must be USDC. Legacy unparsed `transfer` instructions are rejected because mint cannot be inferred.

Output: Hardened `verifySolanaPayment`, four integration tests covering the failure matrix, REVIEW.md HIGH-07 and HIGH-10 marked FIXED.
</objective>
