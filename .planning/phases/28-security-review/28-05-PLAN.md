<objective>
Remediate HIGH-11 by hardening `POST /wallet/export-key` with four guardrails: (1) audit row per attempt, (2) per-user rate limit (3/hour), (3) recent re-auth requirement (fresh JWT within 5 minutes), (4) out-of-band email confirmation on successful export.

Purpose: Any bearer-token leak (XSS, browser extension, leaked Referer) currently equals immediate wallet drain. Layering defenses turns leak → drain into leak → detected-and-rate-limited → email-notified.

Output: Migration for audit table, hardened endpoint, tests covering each new gate, REVIEW.md marked FIXED.
</objective>
