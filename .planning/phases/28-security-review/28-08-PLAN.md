<objective>
Remediate HIGH-02 (Twitter OAuth) with four interlocking fixes:
1. Add `state` nonce on OAuth init.
2. Move pending OAuth state from in-memory `oauthRequests` Map to `x402_oauth_pending` Postgres table with 10-min TTL.
3. Verify `state` on callback (reject missing/expired/mismatched).
4. Encrypt `access_token` and `access_secret` at rest in `x402_user_x_tokens` using existing `encryptSecret` from `lib/instant/encrypt`.

Purpose: Current flow is hijack-prone (no CSRF state), leaks memory (Map never swept across process restarts), and stores tokens plaintext (DB-read compromise = account-takeover supply). Reuse Phase 27's ciphertext-column-then-drop pattern.

Output: New migration, hardened OAuth handlers, re-encryption script, REVIEW.md FIXED. autonomous: false — re-encryption of existing tokens needs operator sanity check before plaintext columns are eventually dropped.
</objective>
