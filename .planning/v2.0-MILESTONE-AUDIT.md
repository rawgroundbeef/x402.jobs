### Previous Gap Resolution

**Gap:** Link Existing publish failed because `TYPE_TO_API` mapped `link → "external"` but `/resources/instant` API didn't accept "external".

**Fix (Phase 26):** Review page now routes link type to `POST /resources` (external registration endpoint, line 78 in review/page.tsx) instead of `POST /resources/instant`. Full verification data (payTo, maxAmountRequired, asset, mimeType, etc.) saved to linkConfig during validation step and passed through to API. Backend endpoint at line 1430 in resources.ts accepts all required fields.
